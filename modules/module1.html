🤔

❓ If you could design a system to solve this, what would you want it to do?

How often do you find no seats at the library?

💡 Think: How many classmates face this too? When a problem affects most people, it's worth solving with tech. Now let's learn to define the problem like an engineer.

Engineer's Perspective

There's no single right order, but engineers usually think like this:

1
Count People
Data comes first. Sensors are the foundation of everything.
2
Display Info
Data is useless unless users can understand it. Visualization creates value.
3
Predicted Times
Predictions need historical data. This is the most advanced feature.

This maps perfectly to our learning path 👇

① Problem Def. → ② Decomposition → ③ Pattern Recog. → ④ Abstraction → ⑤ Algorithm Des.
