🎨 Visual Designer — Data to Interface

~35 min
Objective: Understand abstraction — keep key info, discard irrelevant details, turn raw sensor data into simple visual interfaces so users can grasp library crowding at a glance.

Continuing from Module 3

In Module 3 you discovered peaks, troughs, and abnormal spikes in weekly data.
But if you handed raw data tables to students, could they understand at a glance?

Which approach best helps students quickly decide "should I go to the library now"?

💡 Key Insight: There is no single right answer, but traffic lights and heatmaps best fit "understand at a glance" — they abstract complex data into color semantics: red=crowded, green=free. This is the power of abstraction: keep key info (crowding level), discard irrelevant detail (exact count).

Design Self-Checklist

Check your design — not just "is it there?" but "is it good?"

Readability
Can users grasp key info in 3 seconds?
Think: is the most important number the biggest?
Uses color semantics (red=crowded, green=free)?
Avoid numbers alone — numbers require reading, color requires only perception.
Mobile-Friendly
Is info still clear on a small phone screen?
Students check on the metro one-handed. Can your design handle that?
Does it avoid information overload?
Less is more — abstraction discards unnecessary details.
Feature Complete
Includes total count, floor occupancy, and recommended times?
Three things users most want to know.
Does your design address the user's core need?
Core need = "Should I go to the library now?" — Can your interface help decide?
